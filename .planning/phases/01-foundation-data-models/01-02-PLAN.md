Create the frontend directory using create-next-app, then configure for the project.

1. Run `npx create-next-app@latest frontend --typescript --tailwind --eslint --app --src-dir --import-alias "@/*" --use-npm` from project root
   - This creates Next.js 15 with App Router, TypeScript, Tailwind CSS

2. Verify/update `frontend/tsconfig.json` to ensure:
   - "strict": true
   - "baseUrl": "."
   - "paths": { "@/*": ["./src/*"] }

3. Update `frontend/src/app/layout.tsx` to:
   - Use Inter font from next/font/google
   - Set metadata: title="Loan Document Extraction", description="Extract borrower data from loan documents"
   - Include proper html lang="en" and body with className

4. Update `frontend/src/app/page.tsx` to show a landing page:
   - Centered content with project title "Loan Document Extraction"
   - Subtitle text explaining the system
   - Basic Tailwind styling (min-h-screen, flex, items-center, justify-center)
   - A "Get Started" text/placeholder (button will be added with shadcn/ui later)

5. Verify `frontend/src/app/globals.css` has Tailwind directives:
   - @tailwind base;
   - @tailwind components;
   - @tailwind utilities;

AVOID: Using Tailwind v4 (stick with v3 from create-next-app for shadcn/ui compatibility). Using Pages Router (use App Router).
  </action>

cd frontend && npm install && npm run build && npm run dev &

  <action>
Initialize shadcn/ui in the frontend project.

1. Run `cd frontend && npx shadcn@latest init` with these options:
   - Style: Default
   - Base color: Slate (or Neutral)
   - CSS variables: Yes
   - React Server Components: Yes
   - Write to components.json: Yes

2. This will:
   - Create `frontend/components.json` with shadcn/ui configuration
   - Create `frontend/src/lib/utils.ts` with cn() utility function
   - Update `frontend/tailwind.config.ts` with shadcn/ui theme extensions
   - Update `frontend/src/app/globals.css` with CSS variables for theming

3. Verify `frontend/components.json` exists and contains:
   - "$schema": "https://ui.shadcn.com/schema.json"
   - "style": "default" (or chosen style)
   - "rsc": true (React Server Components)
   - "aliases" with correct path mappings

4. Verify `frontend/src/lib/utils.ts` exports cn() function:
   - Should use clsx and tailwind-merge
   - Pattern: export function cn(...inputs: ClassValue[]) { return twMerge(clsx(inputs)) }

5. Install one test component to verify setup works:
   - Run `cd frontend && npx shadcn@latest add button`
   - Verify `frontend/src/components/ui/button.tsx` was created

6. Update `frontend/src/app/page.tsx` to use the Button component:
   - Import { Button } from "@/components/ui/button"
   - Replace "Get Started" text with <Button size="lg">Get Started</Button>

AVOID: Using shadcn/ui v0 syntax. Misconfiguring the aliases (must match tsconfig paths).
  </action>

cd frontend && npm run build && npm run dev &
sleep 5 && curl -s http://localhost:3000 | grep -i "button" && pkill -f "next dev"
